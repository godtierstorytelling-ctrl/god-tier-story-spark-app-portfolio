## ğŸ“ Narrative Summary (for recruiters who skim)

1. User enters the app and selects a module.
2. They input story parameters (hero, spark, world, plot, etc.)
3. The frontend sends the data to a secure serverless API route.
4. The server formats the request and sends it to OpenAI.
5. AI returns structured writing data.
6. Story Spark displays it cleanly in the UI.
7. User refines, iterates, and moves through additional modules.
8. The workflow continues until the user has:
- Spark
- Logline
- Hero
- Opposition
- World
- Plot
- Beat Sheet
- Scene Drafts
- Voice Samples
- Daily prompts
9. User exports/saves as needed.
